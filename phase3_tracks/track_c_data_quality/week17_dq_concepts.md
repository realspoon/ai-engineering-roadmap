## 1. 데이터 품질의 6가지 차원

### 1.1 Accuracy (정확성)
- 데이터가 실제 값을 정확하게 나타내는가?
- 예: 고객 전화번호, 판매액
- 측정: 검증 데이터셋과의 비교

### 1.2 Completeness (완전성)
- 모든 필수 데이터가 존재하는가?
- 예: NULL 값, 누락된 필드
- 측정: 비율 = (존재 레코드 / 전체 레코드) × 100

### 1.3 Consistency (일관성)
- 데이터 형식과 값이 일관적인가?
- 예: 날짜 형식, 카테고리 값, 단위
- 측정: 규칙 기반 검증

### 1.4 Timeliness (적시성)
- 데이터가 필요한 시점에 가용한가?
- 예: 실시간 데이터, 배치 지연
- 측정: SLA 기반 모니터링

### 1.5 Validity (유효성)
- 데이터가 정의된 형식과 범위를 만족하는가?
- 예: 타입 검증, 범위 검증
- 측정: 스키마 검증, 데이터 타입 확인

### 1.6 Uniqueness (유일성)
- 중복 데이터가 없는가?
- 예: Primary Key, 고유 제약 조건
- 측정: 중복 비율

## 3. Data Quality Score 계산

### 3.1 가중 점수 모델
```
DQ_Score = (Accuracy × 0.25) +
           (Completeness × 0.25) +
           (Consistency × 0.20) +
           (Timeliness × 0.15) +
           (Validity × 0.10) +
           (Uniqueness × 0.05)
```

### 3.2 점수 해석
- 90-100: Excellent (우수)
- 80-89: Good (양호)
- 70-79: Fair (보통)
- 60-69: Poor (부족)
- 0-59: Critical (위험)
