## 📖 핵심 개념

### 1. ReAct (Reasoning + Acting)
**ReAct(Reasoning + Acting)**는 AI 모델이 복잡한 문제를 해결할 때 **'추론(Reasoning)'**과 **'행동(Acting)'**을 번갈아 가며 수행하도록 유도하는 프롬프트 엔지니어링 기법입니다.

단순히 질문에 답하는 것을 넘어, 모델이 스스로 계획을 세우고 외부 도구(검색, 계산기 등)를 사용해 정보를 확인하며 답을 찾아가는 과정을 포함합니다.

---

### 1.1. ReAct의 핵심 구성 요소

ReAct 프롬프트는 보통 **[Thought - Action - Observation]**의 반복 루프로 구성됩니다.

* **Thought (추론):** 모델이 현재 상황을 분석하고, 다음 단계를 결정하기 위해 스스로 생각하는 과정입니다. (예: "이 질문에 답하려면 먼저 A를 검색해야겠다.")
* **Action (행동):** 결정을 바탕으로 실제 동작을 수행합니다. 주로 외부 API 호출이나 데이터베이스 검색 등이 해당됩니다. (예: `search[A에 대한 정보]`)
* **Observation (관찰):** 행동의 결과로 얻은 정보를 확인합니다. (예: 검색 결과 내용)

이 과정을 거치며 모델은 자신의 오류를 수정하거나 부족한 정보를 보완하여 최종 답변에 도달합니다.

---

### 1.2. 왜 ReAct를 사용하나요?

기존의 기법들과 비교했을 때 다음과 같은 확실한 장점이 있습니다.

| 기법 | 특징 | 한계점 |
| --- | --- | --- |
| **Standard Prompt** | 질문에 즉각 답함 | 복잡한 논리나 최신 정보 부족 시 환각(Hallucination) 발생 |
| **Chain of Thought (CoT)** | 단계별 사고 과정을 출력 | 외부 세계와 단절되어 있어 잘못된 사실을 바탕으로 논리만 펼칠 수 있음 |
| **ReAct** | **사고 + 외부 도구 활용** | CoT의 논리성과 실제 데이터의 정확성을 결합해 신뢰도가 높음 |

---

### 1.3. ReAct의 작동 예시

사용자가 **"2024년 올림픽 금메달 순위 1위 국가의 수도는 어디야?"**라고 물었을 때의 과정입니다.

1. **Thought:** 2024년 올림픽 금메달 순위 1위 국가가 어디인지 알아야 한다.
2. **Action:** `search[2024 올림픽 금메달 순위 1위]`
3. **Observation:** 미국이 금메달 40개로 1위를 기록했다.
4. **Thought:** 1위 국가는 미국이다. 이제 미국의 수도가 어디인지 확인해야 한다.
5. **Action:** `search[미국의 수도]`
6. **Observation:** 미국의 수도는 워싱턴 D.C.이다.
7. **Final Answer:** 2024년 올림픽 금메달 1위 국가는 미국이며, 그 수도는 워싱턴 D.C.입니다.

---

### 1.4. ReAct의 한계와 주의점

* **비용과 속도:** 외부 도구를 여러 번 호출하고 사고 과정을 길게 출력하기 때문에 토큰 소모가 크고 답변 속도가 느려질 수 있습니다.
* **무한 루프:** 모델이 적절한 Action을 찾지 못하면 계속해서 같은 행동을 반복할 위험이 있습니다.

---

ReAct는 특히 **LangChain** 같은 LLM 오케스트레이션 프레임워크에서 에이전트(Agent)를 구현할 때 가장 기본이 되는 논리 구조입니다.

### 2. Self-Consistency

**Self-Consistency(자기 일관성)**는 AI가 더 정확한 답을 내놓을 수 있도록 돕는 프롬프트 기법입니다.

쉽게 비유하자면, **"중요한 결정을 내릴 때 혼자 고민하지 않고, 여러 번 스스로 생각한 뒤 가장 많이 나온 의견(다수결)을 채택하는 방식"**이라고 이해하시면 됩니다.

---

### 2.1. 왜 Self-Consistency가 필요한가요?

LLM(거대 언어 모델)은 가끔 첫 번째 시도에서 논리적 실수를 범할 때가 있습니다. 하지만 **Chain of Thought (CoT, 단계별 사고)**를 적용해 여러 번 추론하게 하면, 대부분의 경로에서 올바른 정답에 도달할 확률이 높아집니다.

Self-Consistency는 이 '다양한 사고 경로' 중 가장 일관되게 나타나는 결과가 정답일 가능성이 크다는 원리를 이용합니다.

---

### 2.2. 작동 프로세스

1. **프롬프트 입력:** 질문과 함께 "단계별로 생각해보자"라는 지시를 줍니다.
2. **샘플링 (여러 번 실행):** 모델이 동일한 질문에 대해 서로 다른 추론 과정을 거쳐 여러 개의 답변을 생성하게 합니다.
3. **다수결 (Majority Voting):** 도출된 답변들 중 가장 빈번하게 등장한 답을 최종 정답으로 선택합니다.

---

### 2.3. 구체적인 예시 (수학 문제)

**질문:** "사과가 10개 있습니다. 친구에게 3개를 주고, 시장에서 5개를 더 샀습니다. 그중 2개는 상해서 버렸습니다. 남은 사과는 몇 개입니까?"

이 질문을 모델에게 3번 던졌다고 가정해 봅시다.

* **시도 1 (오류 발생):** "10개에서 3개를 주면 7개, 5개를 사면 12개, 그런데 상한 걸 잊었네요. 답은 **12개**입니다."
* **시도 2 (정답):** "처음 10개 - 준 것 3개 = 7개. 산 것 5개 더하면 12개. 상한 것 2개 빼면 **10개**. 답은 **10개**입니다."
* **시도 3 (정답):** "10 - 3 + 5 = 12입니다. 여기서 상한 2개를 버리면 10이 남습니다. 답은 **10개**입니다."

**최종 선택:** "10개"가 2번, "12개"가 1번 나왔으므로, 시스템은 **10개**를 최종 답변으로 채택합니다.

---

### 2.4. 코드로 보는 구현 로직 (Python 가상 코드)

실제로 API를 사용할 때는 `n` 파라미터(생성 개수)를 조절하여 구현할 수 있습니다.

```python
import openai
from collections import Counter

def get_self_consistency_answer(question):
    # 1. 동일한 질문에 대해 5개의 서로 다른 추론 결과 생성 (Temperature > 0 권장)
    responses = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[{"role": "user", "content": f"{question} 단계별로 생각해서 답해줘."}],
        n=5, # 5개의 샘플을 생성
        temperature=0.7 # 다양성을 위해 온도를 약간 높임
    )

    answers = []
    for choice in responses.choices:
        # 응답 텍스트에서 최종 숫자나 결론만 추출하는 로직 (가정)
        answer = extract_final_answer(choice.message.content)
        answers.append(answer)

    # 2. 다수결 투표 (Majority Vote)
    most_common_answer = Counter(answers).most_common(1)[0][0]
    
    return most_common_answer

# 실행
question = "사과 10개 중 3개 주고 5개 사고 2개 버리면?"
print(f"최종 합의된 정답: {get_self_consistency_answer(question)}")

```

---

### 2.5. 요약

* **핵심:** 여러 번 물어보고 가장 많이 나온 답을 고른다.
* **장점:** 단순 CoT보다 훨씬 높은 정확도, 특히 수학이나 논리 퀴즈에서 강력함.
* **단점:** API 호출 횟수가 늘어나므로 비용이 더 발생함.

앞의 **ReAct**가 "외부 도구와의 협업"에 집중한다면, **Self-Consistency**는 "내부 논리의 견고함"을 다지는 기법이라고 볼 수 있습니다.

### 3. 프롬프트 최적화 기법

| 기법 | 설명 | 효과 |
|------|------|------|
| **구조화** | XML/JSON 태그 사용 | 명확한 구분 |
| **예시 순서** | 쉬운 것부터 어려운 순 | 학습 효과 향상 |
| **부정형 지시** | "하지 마세요" 보다 "~만 하세요" | 정확도 향상 |
| **출력 제한** | 글자수/형식 지정 | 일관성 확보 |

표에 정리된 네 가지 기법은 AI에게 **'정확한 가이드라인'**을 주어 엉뚱한 대답을 하지 못하게 막는 핵심 전략들입니다. 하나씩 쉽게 풀어서 설명해 드릴게요.

---

### 3.1. 구조화 (XML/JSON 태그 사용)

AI에게 정보를 줄 때 줄글로 길게 쓰는 대신, **태그(`< >`, `[ ]`)**를 사용해 구역을 나눠주는 방식입니다.

* **설명:** 마치 서류함에 라벨을 붙이는 것과 같습니다. "여기서부터 여기까지는 참고 데이터고, 여기서부터는 질문이야"라고 명확히 알려주는 거죠.
* **예시:**
```markdown
<context>
이 문서는 2024년 신제품 사양서입니다.
</context>

<question>
배터리 지속 시간을 알려줘.
</question>

```

* **효과:** AI가 명령(Prompt)과 데이터(Context)를 헷갈리지 않아 답변의 정확도가 올라갑니다.

---

### 3.2. 예시 순서 (쉬운 것부터 어려운 순)

AI에게 예시(Few-shot)를 보여줄 때, 난이도를 점진적으로 높여 배치하는 기법입니다.

* **설명:** 사람을 가르칠 때와 똑같습니다. 사칙연산을 가르칠 때 '1+1'부터 시작해서 '미적분'으로 가는 것과 같은 원리입니다.
* **예시:** 1. (쉬움) 사과 → 과일
2. (중간) 고등어 → 생선
3. (어려움) 고래 → 포유류
* **효과:** AI가 문제의 패턴을 더 논리적으로 파악하게 되어, 복잡한 질문에도 일관된 논리를 유지합니다.

---

### 3.3. 부정형 지시 ("~만 하세요")

"~하지 마세요"라는 금지 명령보다는, **해야 할 일을 명확히 지정**하는 것이 훨씬 효과적입니다.

* **설명:** AI는 의외로 '하지 마'라는 부정어를 처리하는 데 서툽니다. "영어로 말하지 마"라고 하면 '영어'라는 단어에 꽂혀 오히려 영어를 쓸 때가 있죠.
* **예시:**
* (나쁨) "답변에 설명을 길게 하지 마세요."
* (좋음) **"오직 답변 결과만 한 문장으로 출력하세요."**


* **효과:** 모델이 딴길로 새지 않고 목적지에 바로 도달하게 만듭니다.

---

### 3.4. 출력 제한 (글자수/형식 지정)

결과물의 **모양새와 분량**을 미리 정해주는 것입니다.

* **설명:** "보고서 써와"라고 하기보다 "A4 1장 이내, 3줄 요약 포함해서 써와"라고 하면 결과물이 훨씬 깔끔하겠죠?
* **예시:** "답변은 100자 이내로 제한하고, 반드시 `-` 기호를 사용한 불렛 포인트 형식으로 작성해줘."
* **효과:** 데이터 분석이나 자동화 시스템에 AI 답변을 바로 입력해야 할 때, 형식이 틀어져서 오류가 나는 상황을 방지합니다.

### 4. 프롬프트 A/B 테스트

```python
class PromptExperiment:
    def __init__(self, prompts: list[str], test_cases: list[dict]):
        self.prompts = prompts
        self.test_cases = test_cases

    def run(self) -> pd.DataFrame:
        results = []
        for prompt in self.prompts:
            for case in self.test_cases:
                response = llm.invoke(prompt.format(**case['input']))
                score = evaluate(response, case['expected'])
                results.append({
                    'prompt': prompt[:50],
                    'case': case['name'],
                    'score': score
                })
        return pd.DataFrame(results)
```
